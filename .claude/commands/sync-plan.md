# Sync Plan Document

플랜 문서를 실제 구현과 비교하고 구현 기준으로 다시 작성한다.

## Input

사용자가 인수로 플랜 파일 경로를 지정한다:
```
/sync-plan .claude/plans/blog-enhancement.md
```

인수가 없으면 `.claude/plans/`에서 플랜 파일을 검색해 선택하도록 묻는다.

## Analysis Steps

1. **플랜 문서 읽기** — 원래 의도, 파일 대상, 예상 동작 파악.
2. **실제 구현 확인** — `git status --short`, 필요하면 커밋 확인, `git diff`로 실제 변경사항 파악.
3. **수정된 파일 읽기** — 플랜에 언급된 파일의 현재 상태 확인.
4. **플랜 vs 실제 비교** — 다음을 구분:
   - 계획과 정확히 일치한 부분
   - 달라진 부분 (다른 함수명, 파일 경로, 로직 등)
   - 계획 외에 추가된 부분
   - 계획했지만 구현되지 않은 부분

## Rewrite Rules

원본 플랜 파일을 "as-built" 문서로 덮어쓴다:

- 전체 구조 유지 (Background → Files → Implementation → Verification)
- 모든 코드 예시를 실제 구현에 맞게 업데이트
- 계획과 다르게 구현된 부분에 `> ⚠️ 계획과 다르게 구현된 부분` 블록쿼트 추가
- 결국 불필요했던 섹션 제거
- 플랜에 없던 중요한 추가 사항은 새 섹션으로 추가
- Notion DB 의존성, env var 요구사항, 빌드 명령어를 정확하게 유지

## Output

- 플랜 파일을 in-place로 재작성
- 재작성 후 플랜과 실제 구현 간 주요 차이점 요약 출력
